# AGENTS.md

Guía operativa para agentes de código (Codex, Cursor y otros) en este repositorio.
Fuente de verdad completa: `CLAUDE.md`. Toda regla nueva se escribe primero allá y se refleja aquí.

**Proyecto:** CRM Pulse

---

## Reglas Absolutas (resumen)

1. **MODO PLAN**: tareas de 3+ pasos o con decisión de arquitectura → plan **en pantalla** (no
   archivo) y esperar **GO** del dueño antes de implementar. Si algo se rompe, PARAR y replanificar.
2. **BUCLE DE MEJORA CONTINUA**: todo error de sesión se registra en `.claude/memory/gotchas.md`
   (fecha, categoría, causa raíz, regla preventiva). Con 3+ ocurrencias se promueve a «Errores
   Críticos» en `CLAUDE.md`. Leer `MEMORY.md` + `gotchas.md` al iniciar sesión.
3. **VERIFICACIÓN ANTES DE DAR POR HECHO**: nada se marca completado sin prueba verificable
   (typecheck, build, tests y flujo real en navegador para UI).
4. **IDIOMA**: prosa, commits, docs y dominio de producto en **español**; nombres de framework/API
   en su forma original (inglés). Identificadores de dominio en español (`cliente`, `seguimiento`).
5. **SIN REFERENCIAS A IA**: sin "Co-Authored-By", sin "Generated by" ni atribuciones de IA en
   commits, código, docs o PRs. El trabajo se firma como del dueño.
6. **POLÍTICA DE DECISIONES**: stack, arquitectura y dependencias mayores las decide el dueño; el
   agente presenta opciones con pros y contras.
7. **CORREGIR LO PROPIO, REPORTAR LO PREEXISTENTE**: fallos introducidos por el cambio se corrigen
   dentro del cambio; la deuda preexistente se reporta sin ampliar el alcance.
8. **FLUJO GIT — `master`**: rama de trabajo y de deploy = `master` (Railway auto-deploya; NO hay
   `main` ni PRs). Commits en Conventional Commits. Ramas `feat/*` opcionales con merge a `master`.
9. **NUNCA PUSH SIN APROBACIÓN**: `git push origin master` = deploy a producción. Commits locales
   sí; push requiere confirmación del dueño.
10. **VALIDAR ANTES DE PUSH**: correr los Quality Gates localmente (typecheck, lint, build, tests,
    `npx convex dev --once`). Push = deploy: no subir lo que vaya a fallar.
11. **VERSIONES**: sin downgrades sin aprobación; preferir la última estable PERO respetar los pins
    del ecosistema (p. ej. Convex Auth ancla `@auth/core` ~`0.41.1`).
12. **SIN SECRETS EN EL REPO**: credenciales solo en `.env.local` (gitignored) o en las variables
    del deployment (Convex/Railway). Secreto filtrado = rotar de inmediato.
13. **AUTORIZACIÓN EN CADA FUNCIÓN (Convex, NO RLS)**: toda `query`/`mutation` valida identidad y
    autoriza dentro de sí misma (`requireUsuario`/`resolverUsuario`); nunca confiar en el cliente;
    validadores en `convex/schema.ts`; fallar **cerrado**.
14. **VERIFICAR UI DE PLATAFORMAS ANTES DE INSTRUIR**: no dictar pasos de dashboards externos
    (Convex/Railway/Linear) desde memoria; verificar con docs oficiales o captura del usuario.
15. **TRABAJO VÍA LINEAR**: issue del equipo `TAL` en "In Progress" antes de tocar código; a "Done"
    al terminar (TAL no tiene "In Review").
16. **PREGUNTAS**: agrupadas dentro de una revisión de plan; sueltas, de una en una.
17. **SKILLS OBLIGATORIOS**: cargar el skill del dominio antes de tocar su superficie (tabla en
    `CLAUDE.md`: `verify`, `run`, `code-review`, `simplify`, `security-review`).

## Reglas específicas de CRM Pulse

- Fuente de verdad de producto = diseño en `design/PROY CRM Pulse/` (11 pantallas). MVP = pantallas
  al 100%.
- Derivados (estado/valor de cliente, último contacto, total de venta) NO se persisten: se calculan
  en queries.
- Borrado = archivar (soft-delete via `archivado`).
- `convex/_generated` se versiona; tests de Convex excluidos del typecheck (`convex/tsconfig.json`).
- Env-gates dev: `CRM_DEV_USER_EMAIL`, `CRM_ALLOW_SEED` (ausentes en prod → fallan cerrado).

## Errores Críticos — NO Repetir

Espejo de `CLAUDE.md`:

1. **Auth que falla ABIERTA** (`identidad ?? dev`): identidad presente sin usuario → `null`; el
   fallback dev SOLO sin identidad. Fallar cerrado.
2. **Tests de Convex rompen el push**: mantener `**/*.test.ts` excluido en `convex/tsconfig.json`
   (`import.meta.glob`).

## Comandos Esenciales

```bash
# Desarrollo
npm run dev                    # Next.js dev (http://localhost:3000)
npx convex dev                 # Convex en watch

# Validación local (ANTES de push)
npx tsc --noEmit               # typecheck de la app
npm run lint                   # next lint
npm run build                  # next build
npm run test                   # vitest run
npx convex dev --once          # empuja esquema/funciones + typecheck de Convex

# Commit / deploy
git add -A && git commit -m "feat: descripción"   # commits locales OK
git push origin master                            # SOLO con aprobación → Railway auto-deploya
```

## Checklist Pre-Commit / Pre-Deploy

```text
[ ] Issue de Linear en "In Progress" (si es trabajo)
[ ] Typecheck, lint y build pasan localmente
[ ] Tests del cambio en verde; `npx convex dev --once` limpio
[ ] Autorización dentro de cada función Convex nueva (no RLS; falla cerrado)
[ ] No hay secrets en el diff
[ ] UI validada en escritorio y móvil si toca UI
[ ] Commit en Conventional Commits, en español, SIN referencias a IA
[ ] Push a `master` solo con aprobación del dueño
```
